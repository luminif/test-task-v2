## Тесткейсы

1. ### Получение существующего объявления

- Шаги воспроизведения: Отправить ```GET``` запрос на  эндпоинт ```/api/1/item/{id}```, где ```id``` - идентификатор существующего объявления.
- Ожидаемый результат: Сервер вернет ```200 OK```. В теле ответа будут данные объявления.

2. ### Получение несуществующего объявления

- Шаги воспроизведения: Отправить ```GET``` запрос на эндпоинт ```/api/1/item/{id}```, где ```id``` - идентификатор несуществующего объявления.
- Ожидаемый результат: Сервер вернет код ошибки ```404 Not Found```.

3. ### Получение всех объявлений продавца по его идентификатору

- Шаги воспроизведения: Отправить ```GET``` запрос на эндпоинт ```/api/1/{sellerID}/item```, где ```sellerID``` - идентификатор продавца.
- Ожидаемый результат: Сервер вернет ```200 OK``` и все объявления продавца.

4. ### Удаление существующего объявления

- Шаги воспроизведения: Отправить ```DELETE``` запрос на эндпоинт ```/api/2/item/{id}```, где ```id``` - идентификатор существующего объявления.
- Ожидаемый результат: Сервер вернет ```200 OK```.

5. ### Удаление несуществующего объявления

- Шаги воспроизведения: Отправить ```DELETE``` запрос на эндпоинт ```/api/2/item/{id}```, где ```id``` - идентификатор несуществующего объявления.
- Ожидаемый результат: Сервер вернет ```404 Not Found```.

6. ### Создание объявления со всеми обязательными полями

- Шаги воспроизведения: Отправить ```POST``` запрос на эндпоинт ```/api/1/item``` со всеми обязательными полями.
- Ожидаемый результат: Сервер вернет ```200 OK``` и идентификатор объявления.

7. ### Создание объявления без обязательного поля sellerID

- Шаги воспроизведения: Отправить ```POST``` запрос на эндпоинт ```/api/1/item``` без обязательного поля ```sellerID```.
- Ожидаемый результат: Сервер вернет код ошибки ```400 Bad Request```. Поле ```result.message``` в теле ответа должно содержать текст: ```поле sellerID обязательно```.
