## Найденные баги при тестировании микросервиса

### 1. Возможность создания объявления с отрицательными значениями
- Описание: Сервер позволяет создать объявление с отрицательным значением цены, идентификатора продавца, контактов, лайков и просмотров.
- Ожидаемое поведение: Сервер должен возвращать код ошибки ```400 Bad Request```.
- Шаги воспроизведения: Отправить ```POST``` запрос на эндпоинт ```/api/1/item``` с отрицательными значениями полей.
```
{
  "sellerID": -345227,
  "name": "test11",
  "price": -11,
  "statistics": {
    "likes": -100,
    "viewCount": -1337,
    "contacts": -55
  }
}
```
- Фактическое поведение: Сервер создает объявление с отрицательными значениями ```price```, ```sellerId```, ```contacts```, ```likes```, ```viewCount``` и возвращает код ```200 OK``` с телом
```
{
    "status": "Сохранили объявление - UUID"
}
```
Где UUID - идентификатор созданного объявления.
- Серьезность: ```Critical```
- Приоритет: ```High```

### 2. Неправильный тип данных для поля ```price```

- Описание: Поле ```price``` в запросе на создание объявления является целочисленным.
- Ожидаемое поведение: Объявление создается, возвращается код ```200 OK```.
- Шаги воспроизведения: Отправить ```POST``` запрос на эндпоинт ```/api/1/item``` с нецелочисленным значением поля ```price```.
```
{
  "sellerID": 345227,
  "name": "test11",
  "price": 11.1,
  "statistics": {
    "likes": 100,
    "viewCount": 1337,
    "contacts": 55
  }
}
```
- Фактическое поведение: Сервер возвращает код ошибки ```400 Bad Request``` с телом
```
{
    "result": {
        "message": "",
        "messages": {}
    },
    "status": "не передано тело объявления"
}
```
- Серьезность: ```Major```
- Приоритет: ```Medium```

### 3. Статистика по удаленному объявлению остается доступной 
- Описание: Если удалить существующее объявление, то его статистику можно будет просматривать. 
- Ожидаемое поведение: После удаления объявления попытка получить его статистику должна возвращать ```404 Not Found```.
- Шаги воспроизведения: 
	1. Отправить ```DELETE``` запрос на эндпоинт ```/api/2/item/{id}``` с существующим идентификатором объявления.
	2. Отправить ```GET``` запрос на эндпоинт ```/api/1/statistic/{id}``` или ```/api/2/statistic/{id}```.
- Фактическое поведение: Сервер возвращает ```200 OK``` и статистику удаленного объявления. Пример тела:
```
[
    {
        "contacts": 55,
        "likes": 100,
        "viewCount": 1337
    }
]
```
- Серьезность: ```Major```
- Приоритет: ```Medium```

### 4. Отсутствие ограничения на длину поля ```name```
- Описание: Отсутствуют ограничения на максимальную длину поля ```name``` в запросе на создание объявления.
- Ожидаемое поведение: Должно быть ограничение на длину поля ```name```.
- Шаги воспроизведения: Отправить ```POST``` запрос на эндпоинт ```/api/1/item``` с длинным значением поля ```name```.
```
{
  "sellerID": 345227,
  "name": "11111111111111111111111111111111111111111111111111111111111111111111111111111111111111111111111111111111111111111111111111111111111111111111111111111",
  "price": 11,
  "statistics": {
    "likes": 100,
    "viewCount": 1337,
    "contacts": 55
  }
}
```
- Фактическое поведение:  Сервер создает объявление с длинным значением поля ```name``` и возвращает код ```200 OK``` с телом
```
{
    "status": "Сохранили объявление - UUID"
}
```
Где UUID - идентификатор созданного объявления.
- Серьезность: ```Major```
- Приоритет: ```Low```

### 5. Несоответствие статуса в теле ответа: возвращается "status": "500" при HTTP 404 
- Описание: Если удалить несуществующее объявление, сервер вернет код ошибки ```404 Not Found``` со status ```500``` в теле ответа.
- Ожидаемое поведение: Сервер должен возвращать код ошибки ```404 Not Found``` со status ```404``` в теле ответа.
- Шаги воспроизведения: Отправить ```DELETE``` запрос на эндпоинт ```/api/2/item/{id}``` с несуществующим идентификатором объявления.
- Фактическое поведение: Сервер возвращает код ошибки ```404 Not Found``` со status ```500``` с телом
```
{
    "result": {
        "message": "",
        "messages": null
    },
    "status": "500"
}
```
- Серьезность: ```Minor```
- Приоритет: ```Low```
